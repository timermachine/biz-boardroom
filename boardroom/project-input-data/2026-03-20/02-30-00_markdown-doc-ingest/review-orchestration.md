# Rewardz Review Orchestration

## Purpose

This layer defines how the board should be deployed against a problem.
It is separate from attendee identity.
Attendee profiles define who each person is.
Review orchestration defines when to use the full board, subgroups, pairs, or individuals.

## Default Modes

### Full Board

Use the full board when the issue affects overall coherence across proposition, economics, risk, execution, or investor defensibility.

Best for:

- integrated venture review
- funding readiness
- strategic pivots
- decisions that create cross-functional dependencies
- major contradictions between documents or assumptions

### Individual Review

Use an individual attendee for first-pass critique inside their domain.

Expected output:

- strongest objection
- missing evidence
- revision required
- kill criterion

### Pair Review

Use a pair when the issue is driven by a productive tension between two viewpoints.

### Breakout Room

Use a breakout room when the main room is looping, blocked, or carrying too much unresolved detail.
Breakout rooms should return with a narrowed recommendation, explicit trade-offs, and any remaining open questions.

Chris may recommend breakout rooms when deadlock or sequencing problems are visible.

Use when:

- sequencing is unclear
- multiple people are mutually waiting
- the room needs a narrower execution path

Chris should identify who is blocked, who is blocking, what can proceed in parallel, and whether a breakout room is more efficient than continuing in plenary.

## Operating Guidance

- Prefer individual review first for document markup.
- Use pairs to pressure-test tension lines.
- Use trios when a decision spans proposition, execution, and control.
- Use the full board only after narrower reviews have sharpened the issue.
- If a discussion repeats without new evidence or narrowed options, Chris should propose a breakout room.
